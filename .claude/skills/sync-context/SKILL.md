# Sync Context (Pika)

把本次会话的持久化进展同步进记忆与文档，让下个会话无缝续上。**同步的同时always精简**：去重、删过时、压缩啰嗦表述，避免文档越长越占上下文。

## 两类目标
- **Auto-memory**（`C:\Users\saiy\.claude\projects\d--Documents-Projects-Pika\memory\`）：跨会话长期记忆。索引 `MEMORY.md` 每条一行；各文件带 frontmatter（name/description/metadata.type）。**只存非代码可推导的信息**（背景、决策+原因、部署事实、外部资源、用户偏好、进展）。
- **项目文档**（仓库内）：`docs/PROJECT_STATUS.md`（状态+下一步）、`docs/IMPLEMENTATION_PLAN.md`（范围）、`CLAUDE.md`（架构/命令，仅规则变化时改）。

## 步骤
1. 先读现状文件，避免重复/冲突。
2. 提炼值得长期保留的：新决策+原因、部署/环境事实、范围变化、验证结论、下一步。
3. 更新 memory（有则改、无则新建并加索引；旧的错了直接改或删）。
4. 更新 md（已完成移出待办；刷新下一步；删废弃旧方案，只留最新口径）。
5. **精简去重（每次必做）**：
   - 合并讲同一件事的多个 memory 文件；删被合并的冗余文件并更新索引。
   - 删掉代码/git 能直接读出的细节（接口名清单、字段清单、文件清单、逐行步骤）。
   - 压缩来回铺陈的叙述性长句为要点；同一信息只在一处写，别处用链接引用。
6. **安全**：写进仓库的内容不得含真实密钥（只在 NAS `.env`）。
7. 简报：3-5 行说明改了什么、精简了什么、下一步是什么。

## 注意
- 相对日期转绝对日期。
- 没新进展也要做一次精简检查；确无可改就如实说明。
