# Pika 就医服务实施计划（当前）

> 进度以 `PROJECT_STATUS.md` 为准；本文件保留范围与验收口径。

## 当前范围（已完成）

1. `hospital` 全链路贯通（上传/落库/查询/展示）
2. 一份报告多图（`image_paths`）
3. 两阶段上传（草稿预解析 → 人工修正 → 提交）
4. 平台能力：服务市场、成员/角色、个人资料、未登录拦截
5. 公网可用：Azure VM + frp + Caddy，体验版已可触达后端

## 本阶段验收（进行中）

1. 体验版登录/退出登录/重新登录稳定（跨网络重复验证）
2. 未登录时服务页与 AI 页不可进入，且引导到“我”页登录
3. 已登录后资料保存可用（头像 + 昵称）
4. 家人体验成员账号可正常登录并查看个人数据

## 部署要求（维持）

- NAS 挂载：`uploads/medical`、`avatars`、`db` 三个独立卷
- 真实凭据仅在 NAS `.env`（含 `WX_*`、`AZURE_*`、`ADMIN_OPENID`、frp 变量）
- Alembic 管理 schema（`upgrade head`）

## 下一步（按优先级）

1. 完成体验版稳定性回归并固化发布流程
2. 多用户权限控制（他人数据不可删改）
3. 二维码邀请加入
4. 趋势图升级、AI 助手、更多报告类型适配
