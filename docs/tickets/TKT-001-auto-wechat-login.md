# TKT-001｜自动微信登录体验改造

## 基本信息

- ID：`TKT-001`
- 类型：feature
- 优先级：P0
- 状态：doing
- 负责人：待定
- 关联阶段：体验版稳定性回归
- 关联入口：`docs/PROJECT_STATUS.md` / P0-1

## 背景

- 用户反馈 / 现象：当前主要要手动进入“我”页才能触发登录，首屏体验不自然。
- 当前原因判断：登录入口偏后置，缺少启动后的 silent login 主路径。
- 为什么现在做：这是当前体验版放量前最直接的首轮体验问题。

## 目标

1. 小程序启动后优先自动尝试微信登录。
2. 自动登录失败时，能明确回退到手动登录，不让用户卡死。

## 非目标

1. 不重做整套账号体系。
2. 不扩展多账号切换或复杂登录态管理。

## 范围

### 要做
- 启动链路触发 silent login。
- 启动时显示轻量“正在登录中”提示。
- 与现有“我”页手动登录入口打通。

### 不做
- 新的登录页视觉重构。
- 非微信渠道登录。

## 方案备注

- 涉及模块：小程序启动流程、登录态守卫、`/api/auth` 现有链路。
- 关键实现点：自动尝试、幂等保护、失败回退、避免重复弹 toast。
- 风险 / 待确认：真机环境下 silent login 的触发时机与失败反馈节奏。

## 验收口径

1. 首次进入小程序时自动发起登录尝试。
2. 登录成功后无需手动进入“我”页。
3. 登录失败时仍可通过现有手动入口继续完成登录。

## 部署影响

- 前端：需要，登录入口与启动链路会改。
- 后端：无，优先复用现有登录接口。
- 数据库：无。
- 配置 / NAS：无。
