# TKT-003｜上传成功后导航回流优化

## 基本信息

- ID：`TKT-003`
- 类型：feature
- 优先级：P0
- 状态：done
- 负责人：待定
- 关联阶段：体验版稳定性回归
- 关联入口：`docs/PROJECT_STATUS.md` / P0-3

## 背景

- 用户反馈 / 现象：上传提交成功后当前固定进入 report detail，再返回时路径偏深。
- 当前原因判断：成功后的导航策略只考虑查看明细，未考虑回首页或历史页的高频动线。
- 为什么现在做：这是上传闭环最后一步，直接影响连续录入与浏览效率。

## 目标

1. 上传成功后给出更顺手的回流路径。
2. 降低返回栈层级和用户迷失感。

## 非目标

1. 不全面重构全局路由。
2. 不改动报告详情页核心信息结构。

## 范围

### 要做
- 明确成功后默认去向或快捷回流入口。
- 评估“就医首页 / 历史页 / 详情页”三者关系。
- 调整成功页或详情页返回动作。

### 不做
- 全站导航框架重写。
- 与本问题无关的 tab 结构改版。

## 方案备注

- 涉及模块：上传提交成功流、详情页、页面跳转策略。
- 关键实现点：回流目标明确、返回栈简化、与历史页筛选默认值衔接。
- 风险 / 待确认：需要根据真机使用习惯决定默认落点。

## 验收口径

1. 上传成功后用户可低成本回到“就医首页”或“历史页”。
2. 不再出现必须多次返回才能继续下一步的主路径问题。
3. 详情查看需求仍被保留，不牺牲已有查看能力。

## 部署影响

- 前端：需要，导航与页面交互会改。
- 后端：无。
- 数据库：无。
- 配置 / NAS：无。
