# TKT-004｜就医权限默认未向家庭成员全开

## 基本信息

- ID：`TKT-004`
- 类型：bug
- 优先级：P0
- 状态：done
- 负责人：待定
- 关联阶段：体验版稳定性回归
- 关联入口：`docs/PROJECT_STATUS.md` / P0-4

## 背景

- 用户反馈 / 现象：就医服务里，每个人对家庭内其他成员的查看、代上传、编辑、删除权限默认都未打开，需要逐个手动授权。
- 当前原因判断：当前 ACL 基于 owner 显式授权，缺少“同一家庭内默认互相全权限”的初始化或兜底策略。
- 为什么现在做：家庭协作是就医服务主路径，默认关闭会让家人看不到或不能维护彼此记录，容易被理解为功能异常。

## 目标

1. 同一家庭内，默认让每个成员向其他所有活跃家庭成员开放就医服务的全部既有权限。
2. 覆盖已有家庭关系和后续新加入成员，避免用户再逐个开关授权。
3. 保持权限边界只在同一家庭内生效，跨家庭仍严格拒绝。

## 非目标

1. 不重做完整家庭权限模型或 RBAC。
2. 不扩展就医服务以外的模块权限。
3. 不开放跨家庭共享。

## 范围

### 要做
- 明确就医 ACL 默认策略：同一家庭成员之间默认互相开放 `view_report`、`upload_for_owner`、`edit_report`、`delete_report`。
- 处理已有家庭成员之间的默认授权补齐。
- 处理新成员加入家庭后的双向默认授权关系。
- 权限设置页展示应与默认全开策略一致，避免仍呈现为默认关闭。

### 不做
- 完整 RBAC 重构。
- 与就医模块无关的权限体系外延设计。
- 跨家庭授权或公开分享能力。

## 方案备注

- 涉及模块：就医 ACL service/router、家庭成员加入链路、权限设置页、历史/趋势数据可见性。
- 关键实现点：默认授权要幂等；既有数据补齐要可重复执行；新成员加入后要与家庭内所有活跃成员建立双向全权限。
- 风险 / 待确认：`edit_report` 与 `delete_report` 默认开放会扩大同家人操作能力，实施前需确认这是体验版的目标隐私边界。

## 验收口径

1. 已有家庭中，任一成员默认可查看、代上传、编辑、删除其他家庭成员的就医记录。
2. 新成员加入后，系统自动补齐其与家庭内其他活跃成员的双向全部就医权限。
3. 权限设置页默认状态与实际权限一致，不再显示为全部关闭。
4. 跨家庭用户仍无法访问或操作不属于自己家庭的就医记录。

## 部署影响

- 前端：需要，权限设置页默认状态与文案可能会改。
- 后端：需要，默认 ACL 策略与成员加入后的授权补齐会改。
- 数据库：可能需要，若采用持久化补齐或迁移现有授权记录。
- 配置 / NAS：无。
