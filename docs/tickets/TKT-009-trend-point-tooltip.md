# TKT-009｜趋势图点按缺少数值 tooltip

## 基本信息

- ID：`TKT-009`
- 类型：bug
- 优先级：P0
- 状态：done
- 负责人：待定
- 关联阶段：就医浏览体验收口
- 关联入口：`docs/PROJECT_STATUS.md` / P1-4

## 背景

- 用户反馈 / 现象：指标趋势页的趋势图上，用户用手点击某个数据点时，没有 tooltip 展示该点代表的数值。
- 当前原因判断：趋势图当前只完成了折线/点位展示，缺少移动端点按命中、选中态和 tooltip 展示交互。
- 为什么现在做：趋势图点位如果不能直接读数，用户需要改看数据表或凭图估算，影响指标追踪的可用性。

## 目标

1. 用户点击趋势图上的任一数据点时，能看到该点对应的数值信息。
2. tooltip 信息足够明确，至少包含数值、单位和对应日期。
3. 点按交互在真机触摸场景下稳定可用。

## 非目标

1. 不重做趋势图整体视觉设计。
2. 不改动趋势数据查询接口和指标计算口径。
3. 不扩展复杂图表分析能力。

## 范围

### 要做
- 在指标趋势页趋势图中补充数据点点按命中逻辑。
- 展示被点选数据点的 tooltip，包含该点数值、单位、日期等必要信息。
- 提供清晰的选中态或定位反馈，避免用户不知道点中了哪里。
- 处理点位较密、边缘点位、无单位或空数据等基础边界。

### 不做
- 趋势页大规模 UI 重构。
- 新增后端图表接口。
- 替换现有图表实现方案，除非现有实现无法支持基础点按。

## 方案备注

- 涉及模块：小程序指标趋势页、趋势图绘制/触摸事件、tooltip 样式。
- 关键实现点：移动端触摸命中半径要足够友好；tooltip 不能遮挡当前点或溢出图表容器；点位密集时要选择最近点。
- 风险 / 待确认：需要在真机上验证触摸坐标与 canvas / 图表坐标换算是否一致。

## 验收口径

1. 在指标趋势页选择有趋势数据的指标后，点击任一趋势点会显示该点数值 tooltip。
2. tooltip 展示的日期、数值和单位与下方数据表或接口数据一致。
3. 连续点击不同点时，tooltip 能切换到最新点位，不出现错位、残留或遮挡严重的问题。
4. 点击图表空白区域时，不出现错误 tooltip 或页面异常。

## 部署影响

- 前端：需要，趋势页图表交互与样式会改。
- 后端：无，优先复用现有趋势数据。
- 数据库：无。
- 配置 / NAS：无。
