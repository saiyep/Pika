# TKT-011｜指标趋势图横坐标最后日期显示不全

## 基本信息

- ID：`TKT-011`
- 类型：bug
- 优先级：P0
- 状态：todo
- 负责人：待定
- 关联阶段：就医能力升级 / 趋势页体验回归
- 关联入口：`docs/PROJECT_STATUS.md` 对应条目

## 背景

- 用户反馈 / 现象：指标趋势页的趋势图横坐标右侧最后一个日期总是显示不全，截图中右侧日期被截成类似 `2026-05`，无法完整看到日期。
- 当前原因判断：趋势图右侧边距或 X 轴标签对齐方式没有为最后一个日期标签预留足够空间，导致标签超出图表/卡片可视区域后被裁切。
- 为什么现在做：趋势图日期是读懂数据点时间的基础信息，最后一个日期被截断会影响趋势页的可读性和真机观感。

## 目标

1. 指标趋势图横坐标最后一个日期在真机窄屏下完整显示。
2. 保持趋势线、点位、tooltip 和参考区间位置正确，不因修复标签而错位。
3. 不同时间范围和不同日期长度下，首尾日期都不被裁切。

## 非目标

1. 不重做趋势图整体视觉设计。
2. 不改动趋势数据接口、指标计算或时间范围筛选逻辑。
3. 不在本 ticket 内新增图表能力。

## 范围

### 要做
- 检查指标趋势页趋势图的 X 轴标签布局、绘制边距和容器裁切规则。
- 修复右侧最后日期标签显示不全的问题。
- 兼顾左侧首个日期、右侧最后日期、短时间范围、长时间范围等基础边界。
- 在真机或等效窄屏尺寸下验证日期完整显示。

### 不做
- 后端趋势接口改造。
- 新增导出、缩放、拖拽等图表功能。
- 与横坐标裁切无关的趋势页大规模 UI 重构。

## 方案备注

- 涉及模块：小程序指标趋势页、趋势图绘制逻辑、X 轴标签样式/布局。
- 关键实现点：右侧标签可能需要右对齐、增加图表内边距或根据文本宽度动态避让；修复后不能压缩数据点到不可读。
- 风险 / 待确认：不同机型宽度、系统字体和日期格式可能影响标签宽度，需要至少覆盖窄屏真机截图场景。

## 验收口径

1. 打开指标趋势页并查看有趋势数据的指标，横坐标最后一个日期完整可见。
2. 横坐标第一个日期和最后一个日期都不被卡片边界或图表容器裁切。
3. 切换不同时间范围后，日期标签仍完整显示且趋势点位置正确。
4. 点击趋势点 tooltip 时，tooltip 与横坐标标签不出现明显遮挡或错位。
